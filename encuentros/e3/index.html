DAX

CALCULATE(): el repaso de E2

La función estrella de DAX: aplicar una métrica con un filtro a medida

Sintaxis

CALCULATE(
  // expresión a calcular
  SUM( Viajes[Pasajeros] ),
  // filtros que aplicamos
  Rutas[Tipo] = "Urbana"
)

Qué hace

Toma una medida (ej: total de pasajeros) y le aplica un filtro adicional antes de devolver el resultado.

En el ejemplo: cuenta los pasajeros solo de las rutas urbanas, sin importar qué slicer esté activo en la página.

Es la base de todo lo que viene hoy.

DAX

CALCULATE con varios filtros a la vez

Podés sumar tantos filtros como necesites, separados por comas

pasajeros_urbano_tarde =
CALCULATE(
  SUM( Viajes[Pasajeros] ),
  Rutas[Tipo] = "Urbana",
  Viajes[Turno] = "Tarde",
  Calendario[Anio] = 2026
)

Cómo se leen los filtros

Power BI los aplica todos a la vez (AND): el resultado son los pasajeros que cumplen las 3 condiciones simultáneamente.

Cuándo usarlo

Cuando querés un KPI específico (ej: "pasajeros del turno tarde en rutas urbanas en 2026") sin tener que filtrar la página entera.

"CALCULATE multiplica el poder de tu modelo: cada combinación es una nueva historia."
DAX

VAR / RETURN: organizá fórmulas largas

Cuando una fórmula crece, las variables la hacen legible y rápida

Sin VAR · difícil de leer
var_pct =
DIVIDE(
  SUM(Viajes[Pasajeros]) -
  CALCULATE(
    SUM(Viajes[Pasajeros]),
    DATEADD(Calendario[Fecha],-1,MONTH)
  ),
  CALCULATE(SUM(...))
)
→
Con VAR · clara
var_pct =
VAR actual = SUM(Viajes[Pasajeros])
VAR anterior = CALCULATE(
  actual,
  DATEADD(Calendario[Fecha],-1,MONTH)
)
RETURN
  DIVIDE(actual - anterior, anterior)
"VAR no es para escribir fórmulas más cortas, es para que se entiendan al volver a leerlas."
Calendario

¿Por qué necesitás una tabla de calendario?

Sin ella, las funciones de tiempo de DAX no funcionan

El problema

Tu tabla de Viajes tiene una columna Fecha, pero solo con las fechas en que hubo viajes.

¿Qué pasa los días sin operación? ¿Cómo agrupás por semana, trimestre, año?

Sin calendario dedicado: no podés comparar períodos.

La solución

Una tabla aparte con todas las fechas del rango (ej: 2024 al 2027), aunque no haya viajes en algunas.

Se conecta con Viajes por la fecha. Power BI ya sabe qué es enero, qué es trimestre, qué es día hábil.

Habilita TOTALYTD, comparaciones año-a-año y mucho más.

Calendario

Crear la tabla con DAX

Dos opciones: rango fijo (recomendado) o automático

// Opción 1: rango definido por vos (recomendado)
Calendario = CALENDAR( DATE(2024,1,1), DATE(2027,12,31) )

// Opción 2: rango automático según tus datos
Calendario = CALENDARAUTO()

// Después agregás columnas calculadas:
Anio = YEAR( Calendario[Date] )
Mes = MONTH( Calendario[Date] )
NombreMes = FORMAT( Calendario[Date], "mmm" )
Trimestre = "T" & QUARTER( Calendario[Date] )
EsHabil = WEEKDAY( Calendario[Date], 2 ) <= 5

Paso final: click derecho sobre la tabla → "Marcar como tabla de fechas". Sin esto, las funciones de tiempo no funcionan.

DAX · Tiempo

TOTALYTD(): acumulado del año

Year-To-Date: lo que llevamos sumado desde el 1 de enero hasta hoy

pasajeros_ytd =
TOTALYTD(
  SUM( Viajes[Pasajeros] ),
  Calendario[Date]
)

Cuándo usarlo

Para mostrar progreso acumulado: "Llevamos 78.651 pasajeros en 2026", "Cumplimos el 65% del objetivo anual".

Aplicado a ISELIN

KPI de cumplimiento anual de viajes vs. objetivo, ideal para la página de Dirección en el dashboard.

"YTD responde la pregunta: ¿cómo venimos vs. el plan, hasta hoy?"
DAX · Tiempo

SAMEPERIODLASTYEAR()

Comparar con el mismo período del año anterior — sin escribir filtros

pasajeros_anio_anterior =
CALCULATE(
  SUM( Viajes[Pasajeros] ),
  SAMEPERIODLASTYEAR( Calendario[Date] )
)

var_anual = DIVIDE(
  [total_pasajeros] - [pasajeros_anio_anterior],
  [pasajeros_anio_anterior]
)

Lectura típica: "Marzo 2026: +18% vs. marzo 2025". Power BI compara automáticamente cada celda de la matriz contra su equivalente del año pasado, sin que vos escribas un solo filtro.

DAX · Tiempo

DATEADD(): desplazar períodos

Más flexible que SAMEPERIODLASTYEAR — corrés el período X días, meses, trimestres

pasajeros_mes_anterior =
CALCULATE(
  SUM( Viajes[Pasajeros] ),
  DATEADD( Calendario[Date], -1, MONTH )
)

// Tres días atrás:
DATEADD( Calendario[Date], -3, DAY )

// Próximo trimestre:
DATEADD( Calendario[Date], +1, QUARTER )
"DATEADD es la navaja suiza del tiempo en DAX: cualquier comparación de períodos pasa por acá."
DAX

Las funciones "X": cálculos fila por fila

Cuando SUM no alcanza porque hay que multiplicar primero, después sumar

SUMX: el patrón

ingreso_total =
SUMX(
  Viajes,
  Viajes[Pasajeros] * Viajes[Tarifa]
)

Para cada fila de Viajes calcula Pasajeros × Tarifa, y al final suma todo.

AVERAGEX: igual pero promedio

consumo_promedio_viaje =
AVERAGEX(
  Viajes,
  DIVIDE(Viajes[Gasoil], Viajes[Km]) * 100
)

Calcula lts/100km de cada viaje y después promedia. Distinto a sumar gasoil / sumar km.

Ejemplo

Variación porcentual: mes actual vs. anterior

Junta lo aprendido: VAR + CALCULATE + DATEADD

var_pasajeros_mom =
VAR actual = SUM( Viajes[Pasajeros] )
VAR anterior = CALCULATE(
  SUM( Viajes[Pasajeros] ),
  DATEADD( Calendario[Date], -1, MONTH )
)
RETURN
  DIVIDE( actual - anterior, anterior )

Resultado típico en ISELIN abril 2026: +12,9% vs. marzo. Con esa medida en una tarjeta y formato condicional verde/rojo, el director ve el dato sin leer ningún número.

Visualización avanzada

Formato condicional: que el color cuente la historia

Reglas automáticas de color según el valor de cada celda

Cómo se aplica

Click derecho sobre el campo en el visual → Formato condicional → elegir tipo (color de fondo / texto / barra de datos).

Modos: escala (gradiente verde→amarillo→rojo), reglas (umbrales fijos), o por valor de campo.

Ejemplos en ISELIN

  • Matriz Ruta × Turno: fondo rojo si demora > 6 min
  • Tabla de choferes: ícono ⚠️ si siniestralidad > 7%
  • KPI: número en rojo si var. mensual < 0
"El color es el primer mensaje que ve el lector — usalo para guiar la atención."
Visualización avanzada

Tooltips personalizados: detalle al pasar el mouse

Una página oculta del reporte aparece como información emergente

1

Crear una página chica (320×240) y marcarla como "Tooltip"

En propiedades de página → Información sobre herramientas: activar.

2

Diseñarla con la info que querés mostrar al pasar el mouse

Foto del colectivo, KPIs del viaje, mini-gráfico, lo que le suma al lector.

3

En el visual destino, asociar la página tooltip

Propiedades del visual → Información sobre herramientas → tipo: Página de informe → elegir la creada.

"Un buen tooltip es como una tarjeta de presentación: dice lo justo y se va."
Visualización avanzada

Drillthrough: ir al detalle con un click derecho

Llevás al usuario a una página específica, ya filtrada por el dato que clickeó

Cómo se configura

1. Crear la página de detalle (ej: Detalle de Ruta).

2. Arrastrar el campo clave (Rutas[Nombre_Ruta]) a la zona "Obtener detalles" del panel de visualizaciones.

3. En cualquier visual del reporte, click derecho → Profundizar → elegir la página.

Ejemplo ISELIN

Estoy en la página Resumen ejecutivo. Veo en un gráfico que Ruta 6 Malargüe tiene 6,4 min de demora.

Click derecho → Profundizar a Detalle de Ruta → me lleva a una página entera dedicada a Malargüe: viajes, choferes, evolución, siniestros.

Visualización avanzada

Vistas guardadas + botones: navegación pro

Bookmarks capturan un estado del reporte. Botones lo restauran al click.

Caso 1 · Vistas alternativas

Mismo reporte, dos lecturas:

  • Vista mensual: gráficos por mes
  • Vista anual: mismos gráficos, pero por año

Dos botones arriba alternan ambos bookmarks.

Caso 2 · Navegación entre páginas

Botón con acción Navegación de página en lugar de bookmark:

  • Barra lateral con 6 botones, uno por página
  • El usuario navega como si fuera una app
  • Mucho más prolijo que las pestañas estándar
Visualización avanzada

Visuales para mostrar progreso vs. objetivo

Dos opciones nativas de PBI cuando hay meta + actual

KPI · indicador con tendencia

Muestra: valor actual, meta, flecha de tendencia (verde sube / roja baja).

Ideal cuando importa la dirección del cambio, no solo el número.

// Indicador: pct_puntualidad
// Eje de tendencia: Calendario[Mes]
// Objetivo: 0,75

Velocímetro (Gauge)

Aguja de 0 a un máximo, con la meta marcada en la rueda.

Útil para KPIs únicos en la página principal: "Disponibilidad de flota: 85% / meta 90%".

Cuidado: ocupa mucho espacio para mostrar un solo dato. No abusar.

Storytelling

Diseñar para comunicar, no para decorar

4 principios que separan un dashboard útil de uno que se abre dos veces y se olvida

1

Menos es más

Cada gráfico tiene que responder una pregunta. Si no responde nada, sobra.

2

Jerarquía visual

Tamaño y posición indican importancia. Lo más crítico, arriba y grande.

3

Flujo de lectura

Diseñá en Z o F: el ojo barre de izquierda a derecha, de arriba a abajo.

4

Significado consistente

Si el verde es "bueno" en una página, debe serlo en todas.

Storytelling

Paleta de colores: 3-4 tonos y un significado

Si todo es de un color distinto, nada destaca. Reservá colores para mensajes.

#059669

Verde

Indicadores positivos: cumplimiento, puntualidad, disponibilidad.

#DC2626

Rojo

Alertas: demoras, siniestros, outliers que requieren acción.

#EA580C

Naranja

Intermedio: KPIs en zona de atención, valores cerca del umbral.

#475569

Gris

Contexto: bordes, fondos, datos de soporte que no compiten por la atención.

"Los colores no son decoración: son etiquetas que el ojo lee antes que las palabras."

Storytelling

Accesibilidad: que lo entienda cualquiera

No es un nice-to-have: es lo que diferencia un dashboard que se usa de uno que no

Contraste

Texto oscuro sobre fondo claro (o viceversa). Probá con la herramienta WebAIM Contrast Checker: mínimo 4.5:1.

Tamaño de fuente

12pt mínimo en visuales. Títulos 16pt+. Si dudás, hacelo más grande.

Texto alternativo

Cada visual tiene un campo Texto alternativo. Una frase corta para lectores de pantalla y para imprimir el reporte sin gráficos.

"Si tu jefe lo abre desde el celular en un viaje, ¿lo entiende? Si la respuesta es no, rediseñá."
