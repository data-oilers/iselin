Pasos aplicados: la receta queda grabada

Cada cambio se registra en orden y se vuelve a aplicar automáticamente cuando el Excel se actualiza

Así se ve en Power Query

• Origen
• Navegación
• Encabezados promovidos
• Tipo cambiado
• Filas vacías quitadas
• Columna dividida ← editable
• Combinada con Choferes

Cada paso se puede editar, reordenar o eliminar.

Por qué importa

Repetible: el próximo mes pongo el Excel nuevo y todos los pasos se aplican solos.

Auditables: cualquier persona puede ver qué transformaciones se hicieron.

Reversibles: si un paso quedó mal, lo borro sin romper los anteriores.

No tocamos el Excel original: Power Query trabaja con una copia.

"Hacés la limpieza una vez. Power Query la repite para siempre."

Modelo de datos

¿Por qué no pongo todo en una sola tabla?

La diferencia entre un Excel gigante y un modelo de datos bien armado

Una sola tabla gigante — problema

• Se repite el nombre del chofer en cada viaje

• Si un chofer cambia de apellido, hay que corregir 5.000 filas

• Archivo pesado, lento

• Difícil mantener la información consistente

Modelo con tablas conectadas — solución

• Cada chofer aparece una sola vez (tabla Choferes)

• Los viajes sólo guardan el ID del chofer

• Cambios de datos maestros en un único lugar

• Modelo liviano, rápido y mantenible

"En Power BI no se trabaja con una planilla: se trabaja con un modelo."

Buenas prácticas del modelo

Reglas simples que evitan dolores de cabeza más adelante

Hacer ✓

• Usar nombres claros: Viajes, Choferes, no Tabla1

• Una sola tabla de hechos principal

• Tener una tabla de Calendario dedicada

• Relaciones de uno-a-muchos siempre que se pueda

• Documentar qué representa cada tabla

Evitar ×

• Relaciones circulares (A→B→C→A)

• Una sola tabla enorme con todo adentro

• Relaciones muchos-a-muchos sin necesidad

• Nombres en español y en inglés mezclados

• Tablas huérfanas (sin conexión con el resto)

"Un buen modelo se lee como un plano. Un mal modelo, como un nudo."

Dos formas de calcular en Power BI

Cuándo alcanza con arrastrar un campo y cuándo conviene escribir una fórmula

A) Cálculo automático

Arrastrás un campo al visual y Power BI resuelve solo.

Ejemplo: arrastro Km a una tarjeta y muestra la suma.

Rápido, pero poco flexible

B) Medida propia (DAX)

Escribís una fórmula que Power BI calcula de forma dinámica según los filtros del reporte.

Ejemplo: Total Viajes, % de cumplimiento, variación mes anterior.

Potente, reutilizable, profesional

"Las medidas son el cerebro de un dashboard profesional."
DAX

Las 6 funciones básicas

Con estas seis resolvés la gran mayoría de los KPIs

SUM()

Suma una columna numérica

Total Km = SUM(Viajes[Km])

COUNT()

Cuenta filas con valor

Total Viajes = COUNT(Viajes[ID_Viaje])

AVERAGE()

Promedio de una columna

Km Promedio = AVERAGE(Viajes[Km])

DISTINCTCOUNT()

Cuenta valores únicos

Choferes Activos = DISTINCTCOUNT(Viajes[ID_Chofer])

MIN() / MAX()

Valor mínimo o máximo

Viaje Más Largo = MAX(Viajes[Km])

DIVIDE()

Divide y maneja el error de división por cero

Km/Viaje = DIVIDE([Total Km], [Total Viajes])

"Dominar estas seis te deja construir casi cualquier KPI de ISELIN."
DAX

Anatomía de una medida

Cada pieza tiene un rol específico

Total Km = SUM ( Viajes [ Km ] )
NombreCómo la vas a ver en el panel de campos. Usá nombres claros y en tu idioma.
FunciónQué operación hace. Aquí SUM suma valores.
TablaDe qué tabla saca los datos. Siempre entre comillas si tiene espacios.
ColumnaQué columna específica usa. Siempre entre corchetes.
"Una medida bien nombrada se entiende sin necesidad de leer la fórmula."
DAX

CALCULATE(): la función estrella

Hace el mismo cálculo de siempre, pero aplicando un filtro extra

La idea

CALCULATE toma una medida y le dice:

"Calculá esto, pero sólo para estas filas."

Sintaxis

CALCULATE( expresión, filtro1, filtro2, ... )

-- Total de viajes en la Ruta 5
Viajes Ruta 5 =
  CALCULATE(
    COUNT( Viajes[ID_Viaje] ),
    Rutas[Nombre] = "Ruta 5"
  )
"Si alguna vez escribiste SUMAR.SI en Excel, ya sabes lo que hace CALCULATE."
DAX

Tres medidas para el dashboard de ISELIN

Casos reales que vamos a construir en la demo

-- Cuántos viajes hicimos en total
Total Viajes = COUNT( Viajes[ID_Viaje] )
-- Kilómetros promedio por chofer
Km Prom Chofer = DIVIDE( SUM(Viajes[Km]), DISTINCTCOUNT(Viajes[ID_Chofer]) )
-- Viajes del mes actual
Viajes Mes Actual =
  CALCULATE(
    [Total Viajes],
    Calendario[Mes] = MONTH( TODAY() )
  )
"Las medidas se reutilizan: una vez creadas, se combinan entre sí en otras fórmulas."

Segmentadores (slicers): filtros interactivos

El usuario elige y el dashboard entero se actualiza

Cómo se agrega

1. En el panel de visualizaciones, clic en el ícono de Segmentación

2. Arrastrá el campo al segmentador (Fecha, Ruta, Chofer...)

3. Elegi el estilo: lista, desplegable, rango de fechas

4. Los demás visuales se filtran en conjunto

Segmentadores recomendados para ISELIN

• Rango de fechas — filtrar por mes o período

• Ruta — aislar una ruta específica

• Base / Agencia — ver sólo una localidad

• Tipo de servicio — urbano, larga distancia, etc.

"Un dashboard con buenos slicers responde decenas de preguntas con la misma página."
Visualizaciones

Formato básico: consistencia antes que decoración

Un dashboard ordenado se lee más rápido — y genera más confianza

Colores

Máximo 3-4 colores por dashboard. Usá la paleta corporativa de ISELIN.

Verde = bueno, rojo = alerta. Que el color siempre signifique lo mismo.

Títulos

Un título descriptivo en cada visual: "Viajes por ruta · Abril 2026".

Evitá "Gráfico 1" o "Tabla nueva".

Etiquetas y ejes

Mostrá etiquetas de datos en los gráficos clave. Formato métrico: "2.500 km", no "2500".

Separadores de miles — siempre.

"Menos es más. Cada elemento del dashboard debe justificar su presencia."
